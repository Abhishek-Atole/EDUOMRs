## 1. Database Principles

| Principle | Guideline |
|---|---|
| Database | PostgreSQL 16+ |
| Primary keys | UUID v4 (generated by gen_random_uuid()) |
| Timestamps | All tables: created_at, updated_at (auto), deleted_at (soft delete) |
| Tenancy | All tenant-scoped tables include tenant_id as NOT NULL |
| Naming | snake_case for tables, columns, indexes, constraints |
| Migrations | Prisma Migrate — versioned, reviewed, reversible |
| Constraints | Foreign keys with CASCADE for referenced deletes where appropriate |
| Indexes | Index on tenant_id, foreign keys, status, created_at for common queries |

## 3. Complete Table Schemas

### 3.1 institutions

```sql
CREATE TABLE institutions (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name            VARCHAR(255) NOT NULL,
  address         TEXT,
  contact_email   VARCHAR(255),
  contact_phone   VARCHAR(20),
  status          VARCHAR(20) NOT NULL DEFAULT 'pending'
                  CHECK (status IN ('pending', 'active', 'disabled')),
  settings        JSONB DEFAULT '{}',
  created_at      TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  updated_at      TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  deleted_at      TIMESTAMP WITH TIME ZONE
);

CREATE INDEX idx_institutions_status ON institutions(status) WHERE deleted_at IS NULL;
CREATE INDEX idx_institutions_name ON institutions(name);
```

### 3.2 users

```sql
CREATE TABLE users (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  tenant_id       UUID NOT NULL REFERENCES institutions(id) ON DELETE CASCADE,
  email           VARCHAR(255) NOT NULL,
  password_hash   VARCHAR(255) NOT NULL,
  first_name      VARCHAR(100) NOT NULL,
  last_name       VARCHAR(100),
  phone           VARCHAR(20),
  role            VARCHAR(20) NOT NULL
                  CHECK (role IN ('platform_owner', 'super_admin', 'admin', 'teacher', 'student', 'parent')),
  is_active       BOOLEAN DEFAULT true,
  failed_attempts INTEGER DEFAULT 0,
  locked_until    TIMESTAMP WITH TIME ZONE,
  created_at      TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  updated_at      TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  deleted_at      TIMESTAMP WITH TIME ZONE,

  UNIQUE(tenant_id, email)  -- email unique within tenant
);

CREATE INDEX idx_users_tenant_role ON users(tenant_id, role) WHERE deleted_at IS NULL;
CREATE INDEX idx_users_email ON users(email) WHERE deleted_at IS NULL;
CREATE INDEX idx_users_tenant ON users(tenant_id) WHERE deleted_at IS NULL;
```

### 3.3 parent_students

```sql
CREATE TABLE parent_students (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  tenant_id       UUID NOT NULL REFERENCES institutions(id) ON DELETE CASCADE,
  parent_id       UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  student_id      UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  created_at      TIMESTAMP WITH TIME ZONE DEFAULT NOW(),

  UNIQUE(parent_id, student_id),
  CONSTRAINT chk_parent_role CHECK (EXISTS (SELECT 1 FROM users WHERE id = parent_id AND role = 'parent')),
  CONSTRAINT chk_student_role CHECK (EXISTS (SELECT 1 FROM users WHERE id = student_id AND role = 'student'))
);

CREATE INDEX idx_parent_students_parent ON parent_students(parent_id);
CREATE INDEX idx_parent_students_student ON parent_students(student_id);
```

### 3.8 classes, sections, subjects

```sql
CREATE TABLE classes (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  tenant_id       UUID NOT NULL REFERENCES institutions(id) ON DELETE CASCADE,
  name            VARCHAR(100) NOT NULL,
  academic_year_id UUID REFERENCES academic_years(id),
  created_at      TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  updated_at      TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  deleted_at      TIMESTAMP WITH TIME ZONE,

  UNIQUE(tenant_id, name, academic_year_id)
);

CREATE TABLE sections (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  tenant_id       UUID NOT NULL REFERENCES institutions(id) ON DELETE CASCADE,
  class_id        UUID NOT NULL REFERENCES classes(id) ON DELETE CASCADE,
  name            VARCHAR(100) NOT NULL,
  created_at      TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  updated_at      TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  deleted_at      TIMESTAMP WITH TIME ZONE,

  UNIQUE(tenant_id, class_id, name)
);

CREATE TABLE subjects (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  tenant_id       UUID NOT NULL REFERENCES institutions(id) ON DELETE CASCADE,
  class_id        UUID NOT NULL REFERENCES classes(id) ON DELETE CASCADE,
  name            VARCHAR(100) NOT NULL,
  code            VARCHAR(20),
  created_at      TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  updated_at      TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  deleted_at      TIMESTAMP WITH TIME ZONE,

  UNIQUE(tenant_id, class_id, name)
);

CREATE INDEX idx_classes_tenant ON classes(tenant_id);
CREATE INDEX idx_sections_class ON sections(class_id);
CREATE INDEX idx_subjects_class ON subjects(class_id);
```

### 3.10 exams

```sql
CREATE TABLE exams (
  id                UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  tenant_id         UUID NOT NULL REFERENCES institutions(id) ON DELETE CASCADE,
  title             VARCHAR(255) NOT NULL,
  description       TEXT,
  exam_mode         VARCHAR(20) NOT NULL
                    CHECK (exam_mode IN ('DIGITAL', 'PHYSICAL_PAPER')),
  total_marks       INTEGER NOT NULL,
  marks_per_correct DECIMAL(5,2) NOT NULL DEFAULT 1.00,
  marks_per_wrong   DECIMAL(5,2) NOT NULL DEFAULT 0.00,
  negative_marking  BOOLEAN DEFAULT false,
  duration_minutes  INTEGER NOT NULL,
  status            VARCHAR(20) NOT NULL DEFAULT 'draft'
                    CHECK (status IN ('draft', 'published', 'in_progress', 'completed', 'results_released')),
  scheduled_at      TIMESTAMP WITH TIME ZONE,
  deadline_at       TIMESTAMP WITH TIME ZONE,
  class_id          UUID REFERENCES classes(id),
  subject_id        UUID REFERENCES subjects(id),
  created_by        UUID REFERENCES users(id),
  created_at        TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  updated_at        TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  deleted_at        TIMESTAMP WITH TIME ZONE
);

CREATE INDEX idx_exams_tenant_status ON exams(tenant_id, status) WHERE deleted_at IS NULL;
CREATE INDEX idx_exams_class ON exams(class_id) WHERE deleted_at IS NULL;
CREATE INDEX idx_exams_scheduled ON exams(scheduled_at) WHERE status = 'published';
```

### 3.11 questions

```sql
CREATE TABLE questions (

  correct_option  VARCHAR(1) NOT NULL CHECK (correct_option IN ('A', 'B', 'C', 'D')),
  marks           DECIMAL(5,2) NOT NULL DEFAULT 1.00,
  order_index     INTEGER NOT NULL,
  created_at      TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  updated_at      TIMESTAMP WITH TIME ZONE DEFAULT NOW(),

  UNIQUE(exam_id, order_index)
);

CREATE INDEX idx_questions_exam ON questions(exam_id);
```

## 4. Index Strategy Summary

| Table | Index | Type | Rationale |
|---|---|---|---|
| institutions | status | B-tree | Filter active/pending/disabled |
| users | (tenant_id, role) | Composite | List users by institution + role |
| users | email | B-tree | Login lookup |
| exams | (tenant_id, status) | Composite | Dashboard list by status |
| exams | scheduled_at | B-tree | Auto-publish job, upcoming exams |
| exam_sessions | (status, exam_id) | Partial | Auto-submit job finds in-progress |
| results | (exam_id, percentage DESC) | Composite | Rank calculation |
| notification_logs | status | B-tree | Dead-letter queue monitoring |
| refresh_tokens | token_hash | B-tree | Token lookup on refresh |

## 5. Migration Strategy

### 5.1 Principles

- All schema changes via Prisma Migrate
- Migrations are versioned and stored in `prisma/migrations/`
- Backward-compatible changes only (add columns, not remove)
- Destructive changes require explicit approval + data backup
- Production migrations run during maintenance window

### 5.2 Prisma Schema

```prisma
generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}

model Institution {
  id            String   @id @default(uuid()) @db.Uuid
  name          String   @db.VarChar(255)
  address       String?  @db.Text
  contactEmail  String?  @map("contact_email") @db.VarChar(255)
  contactPhone  String?  @map("contact_phone") @db.VarChar(20)
  status        String   @default("pending") @db.VarChar(20)
  settings      Json?    @default("{}")
  createdAt     DateTime @default(now()) @map("created_at")
  updatedAt     DateTime @updatedAt @map("updated_at")
  deletedAt     DateTime? @map("deleted_at")

  users             User[]
  subscriptions     Subscription[]
  paymentUploads    PaymentUpload[]
  academicYears     AcademicYear[]
  classes           Class[]
  enrollments       Enrollment[]
  exams             Exam[]
  notificationLogs  NotificationLog[]
  auditLogs         AuditLog[]
  refreshTokens     RefreshToken[]
  passwordResetTokens PasswordResetToken[]

  @@map("institutions")
}

// ... remaining models follow the same pattern
```

### 5.3 Migration Workflow

```
1. Engineer creates feature branch
2. Modifies Prisma schema
3. Runs `npx prisma migrate dev --name migration_name`
4. Migration SQL file generated + reviewed in PR
5. PR approved and merged
6. CI/CD runs `npx prisma migrate deploy` on staging
7. Staging tests pass
8. CI/CD runs `npx prisma migrate deploy` on production
```

## 6. Query Patterns

### 6.1 Tenant-Scoped Query (Repository)

```javascript
async function findByTenant(tenantId, filters = {}) {
  return prisma.exam.findMany({
    where: {
      tenant_id: tenantId,
      deleted_at: null,
      ...filters
    },
    orderBy: { created_at: 'desc' },
    take: 20,
    skip: 0
  });
}
```

### 6.2 Student Exam Session with Answers

```javascript
async function getSessionWithAnswers(sessionId) {
  return prisma.examSession.findUnique({
    where: { id: sessionId },
    include: {
      student_answers: true,
      exam: {
        include: {
          questions: {
            orderBy: { order_index: 'asc' }
          }
        }
      }
    }
  });
}
```

### 6.3 Rank Calculation Query

```sql
SELECT id, percentage,
       RANK() OVER (ORDER BY percentage DESC) as rank,
       COUNT(*) OVER () as total_students
FROM results
WHERE exam_id = $1 AND is_released = true
ORDER BY percentage DESC;
```

### 6.4 Auto-Submit Query

```sql
-- Find all in-progress sessions for deadline-passed exams
SELECT es.* FROM exam_sessions es
JOIN exams e ON es.exam_id = e.id
WHERE es.status = 'in_progress'
  AND e.deadline_at <= NOW();
```

## 7. Data Archival

| Table | Archive Strategy |
|---|---|
| notification_logs | Purge details > 1 year, keep summary |
| audit_logs | Keep indefinitely (append-only, small volume) |
| exam_sessions | Keep indefinitely (exam integrity) |
| student_answers | Keep indefinitely (per-question breakdown) |
| refresh_tokens | Purge revoked tokens > 30 days |
| password_reset_tokens | Purge used/expired tokens > 7 days |
